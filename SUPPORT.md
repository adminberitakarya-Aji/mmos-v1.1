# Support

Thank you for using MMOS (Media & Multimedia Orchestration System).

This document explains how to get help, ask questions, report issues, and contribute to the project.

---

# Getting Help

If you need help using MMOS, please use the appropriate communication channel based on your request.

## Documentation

The first place to look is the project documentation.

Documentation includes:

* Architecture
* Runtime
* Object Model
* Provider Integration
* Examples
* Deployment Guides
* API References

Most common questions are answered in the documentation.

---

# Bug Reports

If you believe you have found a bug:

1. Verify that the issue has not already been reported.
2. Collect logs and relevant information.
3. Include reproduction steps.
4. Open a GitHub Issue using the Bug Report template.

Please include:

* MMOS version
* Operating system
* Runtime environment
* Steps to reproduce
* Expected behavior
* Actual behavior
* Relevant logs (if available)

---

# Feature Requests

We welcome ideas that improve MMOS.

Before submitting a feature request:

* Search existing discussions.
* Explain the problem the feature solves.
* Describe the proposed solution.
* Explain how it aligns with the MMOS architecture.

Large architectural changes should be discussed before implementation.

---

# Questions

For general questions:

* Review the documentation.
* Search existing discussions.
* Open a new discussion if your question has not already been answered.

---

# Security Issues

Please **do not** report security vulnerabilities through public issues.

Instead, follow the instructions described in `SECURITY.md`.
