## Versioning Policy

MMOS follows Semantic Versioning:

* **MAJOR** — incompatible architectural or API changes.
* **MINOR** — new features with backward compatibility.
* **PATCH** — bug fixes and non-breaking improvements.

Example:

* 1.1.0
* 1.2.0
* 1.2.1
* 2.0.0

---

## Release Types

### Major Release

Architectural evolution that may introduce breaking changes.

### Minor Release

New platform capabilities while maintaining compatibility.

### Patch Release

Bug fixes, performance improvements, documentation updates, and internal optimizations without breaking public contracts.
